The Moddable SDK is an open-source development platform that lets you write modern JavaScript applications for microcontrollers — the tiny, low-power chips inside smart appliances, wearables, and connected devices. Instead of learning proprietary embedded languages or complex C toolchains, you use the same JavaScript you already know, backed by a purpose-built runtime that runs on devices with as little as 45 KB of RAM.

## How it works

At the core of the Moddable SDK is the **XS JavaScript engine** — the only JavaScript engine that achieves greater than 99% conformance with the ES2025 standard on microcontrollers. XS compiles your JavaScript to bytecode and stores modules in ROM, minimizing the RAM used at runtime.

You describe your application in a **manifest.json** file that lists your modules, resources, and platform-specific settings. The **mcconfig** build tool reads that manifest, compiles your code, and flashes it to your device (or launches it in the desktop simulator).

<Steps>
  <Step title="Install the SDK">
    Clone the repository, set environment variables, and build the tools on your OS.
  </Step>
  <Step title="Write your app">
    Create JavaScript modules and a `manifest.json` that describes your project.
  </Step>
  <Step title="Build and deploy">
    Run `mcconfig -d -m -p <platform>` to compile and flash your application.
  </Step>
  <Step title="Debug with xsbug">
    Use the xsbug debugger to set breakpoints, inspect variables, and profile memory in real time.
  </Step>
</Steps>

## Supported hardware

The Moddable SDK runs on a wide range of microcontroller families:

| Family | Boards |
|--------|--------|
| **ESP32** | Moddable Two, Moddable Six, M5Stack, NodeMCU ESP32 |
| **ESP8266** | Moddable One, Moddable Three, NodeMCU |
| **nRF52** | Moddable Four, Nordic DK, SparkFun, Adafruit, Seeed XIAO |
| **Raspberry Pi Pico** | Pico W, Pimoroni Pico Display, LILYGO T-Display |
| **Zephyr** | Any Zephyr-supported board |
| **Simulator** | macOS, Linux, Windows desktop |

<Note>
  New to microcontroller development? Start with the desktop simulator — it lets you build and test applications on your computer without any hardware.
</Note>
