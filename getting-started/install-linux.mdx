The Moddable SDK supports Ubuntu 16.04–22.04 LTS (64-bit) and Raspberry Pi Desktop (32-bit). The setup involves installing system dependencies, cloning the repository, and compiling the SDK tools — including the xsbug debugger and the device simulator — from source. The full process typically takes 10–15 minutes.

<Note>
  If you are running Ubuntu inside **WSL2** (Windows Subsystem for Linux), you will need one additional package to support the GTK-based simulator. The relevant step calls this out.
</Note>

## Prerequisites

- Ubuntu 16.04–22.04 LTS (64-bit) or Raspberry Pi Desktop (32-bit)
- `sudo` access to install system packages
- GCC toolchain (installed as part of the steps below)

## Installation

<Steps>
  <Step title="Install required system packages">
    Install the GCC toolchain and build dependencies:

    ```bash
    sudo apt-get install gcc git wget make libncurses-dev flex bison gperf
    sudo apt-get install libgtk-3-dev
    ```

    If you are running Ubuntu under **WSL2**, also install the icon theme required by the simulator:

    ```bash
    sudo apt-get install adwaita-icon-theme-full
    ```
  </Step>

  <Step title="Create a Projects directory">
    ```bash
    mkdir -p ~/Projects
    ```
  </Step>

  <Step title="Clone the Moddable repository">
    ```bash
    cd ~/Projects
    git clone https://github.com/Moddable-OpenSource/moddable
    ```
  </Step>

  <Step title="Configure your shell environment">
    Open `~/.bashrc` in a text editor and add the following lines at the end:

    ```bash
    export MODDABLE=~/Projects/moddable
    export PATH=$PATH:$MODDABLE/build/bin/lin/release
    ```

    Save and close the file.
  </Step>

  <Step title="Apply the environment changes">
    Load the updated configuration into your current session:

    ```bash
    source ~/.bashrc
    ```

    Alternatively, close the terminal and open a new one.
  </Step>

  <Step title="Build the Moddable tools">
    Compile the SDK's native build tools:

    ```bash
    cd $MODDABLE/build/makefiles/lin
    make
    ```

    This step compiles `mcconfig`, `mcrun`, xsbug, and the simulator. It may take several minutes to complete.
  </Step>

  <Step title="Install the simulator and xsbug">
    Run the install target to place the simulator and xsbug in the correct locations for your desktop environment:

    ```bash
    cd $MODDABLE/build/makefiles/lin
    make install
    ```
  </Step>

  <Step title="Launch xsbug">
    Start the xsbug JavaScript debugger to confirm the install succeeded:

    ```bash
    xsbug
    ```

    xsbug should open as a desktop window. Leave it running for the next step.
  </Step>

  <Step title="Run the helloworld verification">
    Build and run the helloworld example on the Linux simulator:

    ```bash
    cd $MODDABLE/examples/helloworld
    mcconfig -d -m -p lin
    ```

    You should see `Hello, World` printed in the xsbug console. A successful run confirms your installation is complete.
  </Step>
</Steps>

## Updating the SDK

When a new version of the Moddable SDK is released, update the GTK dependency, pull the latest source, and rebuild:

```bash
sudo apt-get update && sudo apt-get install --upgrade libgtk-3-dev

cd $MODDABLE
git pull

rm -rf build/tmp build/bin

cd $MODDABLE/build/makefiles/lin
make
make install

cd $MODDABLE/examples/helloworld
mcconfig -d -m -p lin
```

<Tip>
  The `rm -rf build/tmp build/bin` step removes stale compiled artifacts. Skipping it occasionally causes subtle link errors after a pull that updates native tool source files.
</Tip>
